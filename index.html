HTML

  • HTML stands for Hypertext Markup Language.
  • HTML provides the foundation and structure for a website.
  • Elements nested under other elements are called "child" elements; the elements they are nested under are called "parent" elements.
  • The head element contains information (metadata) about the webpage that allow it to render correctly in the browser.
  • The body element represents the visible content shown to the user.
  • Body elements are "semantic" meaning that the name of the element describes the content and the role on the webpage.
  • Proper usage of HTML organization allows improves page accessiblity for users using screen readers or other devices.

CSS

  • There are three ways to style using CSS: inline CSS, internal CSS, and external css.
  • While all are useful, and external CSS allows for consistency and ease.
  • The "selector" is the item that you are defining the characteristic of.
  • To apply stylistic properties to all elements, use the wild card (*) selector.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • You can combine two selectors into a single line, seperated by a comma, to spply style to both selectors (i.e. header, footer)
  • Do Not Repeat Yourself (DRY) coding reduces the lines of coding.
  • Class attributes allow you to pick and choose which HTML elements to apply styling to. To do this, you put a period (.) before the selector. Then any element with that attribute will have that styling.

Git

  • Git is software on your computer that tracks changes made to code.
  • Git allows for increased collaboration amongst coders and version control.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
